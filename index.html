HTML

  • HTML attributes are always defined in the beginning element, never the closing element.
  • Semantic HTML elements describe its content and the role it plays on a webpage.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.

CSS

  • CSS is a programming language used by developers to define how webpage content is presented to users. HTML handles the structure and content, CSS defines the colors, aesthetics, visual layout and organization.
  • Inline CSS: when a developer wants to style an element directly in the HTML file, quick and permanent changes but less flexible because inline must be edited individually.
  • Internal Style Sheet: contains CSS rules for the webpage in the head section of an HTML file. The rules only apply to that page.
  • External CSS Style Sheet: the most common way to style a webpage. It allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • A selector defines the elements or attributes to which the rules, or declarations, will apply.
  • Declarations contain two components: the CSS property we want to apply and the value of that property.
  • Every HTML element is essentially a box.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • The img selector assigns property values to any image element in the markup.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • A class selector is any HTML element assigned to the class will gain the properties from CSS.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it

JavaScript

  • JavaScript is a developer language that adds interactivity to a website.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • Browser Application Programming Interfaces (APIs) are built into web browsers providing functionality.
  • The const keyword allows us to create variables that can't be reassigned a value. It locks down what the variable will be and cannot be reassigned.
  • If Statements will check whether a condition evaluates to be true or false. The answer determines the next course of action and application.
  • An array is a single variable that is used to hold a group of data. Arrays are typically used to hold data that is related in some way.
  • To create an array we declare a variable using const and then give the variable a name and assign it to a group of values.
  • An array index is a unique number that identifies the location of the data item in the array.
  • Arrays in JavaScript are zero-indexed and sequential.
  • A function is a set of instructions that tells the computer how to perform a certain task.
  • Functions do not automatically execute, you must call a function by name in order to execute it.
  • Two requirements for a JavaScript function: 1 Define the function, 2 Call the function.
  • A function's name is always followed by parentheses ().
